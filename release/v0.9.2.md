# @synkro/core v0.9.2

**Release date:** 2026-03-07

## Changes

### TD-13: Harden message parsing paths [SEC]

All `JSON.parse` calls in `HandlerRegistry` and `WorkflowRegistry` are now wrapped with try/catch. Messages that fail to parse or have a missing/invalid `requestId` are logged and dropped instead of crashing processing paths.

- `HandlerRegistry.handleMessage` validates parsed input before proceeding with lock acquisition and handler execution
- `WorkflowRegistry` subscription callbacks use a new `safeParse` helper, rejecting malformed messages before entering lock/transition logic

### IMP-07: Redis connection error handling

`RedisManager` now creates all three Redis connections (publisher, subscriber, cache) via a `createClient` helper that provides:

- **Retry strategy** with exponential backoff capped at 5 seconds
- **Error event handler** that logs connection errors instead of surfacing them as unhandled rejections
- **Connect event handler** for connection status visibility via debug logging
