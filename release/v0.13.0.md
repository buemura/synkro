# @synkro/core v0.13.0

**Release date:** 2026-03-07

## Changes

### IMP-08: Configurable key retention / TTL policy

Redis keys created during event handling and workflow execution now have configurable TTLs via the new `retention` option in `SynkroOptions`. Previously, all TTLs were hardcoded (locks: 5min, dedup/state: 24h, metrics: never expire).

```typescript
const synkro = await Synkro.start({
  transport: "redis",
  connectionUrl: "redis://localhost:6379",
  retention: {
    lockTtl: 60,        // distributed lock TTL in seconds (default: 300)
    dedupTtl: 3600,     // deduplication key TTL in seconds (default: 86400)
    stateTtl: 7200,     // workflow state TTL in seconds (default: 86400)
    metricsTtl: 86400,  // metrics key TTL in seconds (default: undefined / no expiry)
  },
});
```

All fields are optional — omitting `retention` preserves the existing default behavior.

**New type:** `RetentionConfig` is exported from `@synkro/core`.

### TD-07: Configurable workflow state TTL

Workflow state TTL is no longer hardcoded to 24 hours. Use `retention.stateTtl` to configure it per instance. Long-running workflows can now set a higher TTL to prevent state loss.
