# @synkro/next v0.2.0

**Release date:** 2026-03-09

## Changes

### FT-08: Serverless HTTP dispatch — full event & workflow processing in Next.js

The `@synkro/next` package can now **process** events and advance workflows entirely within serverless functions. Previously it could only publish events and serve the dashboard.

**Core idea:** Replace Redis pub/sub listeners (which require persistent connections) with HTTP POST calls to Next.js route handlers. Redis is still used for cache (locking, dedup, state, metrics).

**New exports:**

- **`createSynkroServerless(options)`** — Factory that wires an `HttpTransportManager` + `WorkflowAdvancer` for the serverless use case. Returns `{ client, transport, advancer }`.

- **`createEventHandler(synkro, options)`** — Creates a Next.js route handler (`export const POST`) that processes an event with full dedup, distributed locking, retry, and metrics guarantees.

- **`createWorkflowStepHandler(synkro, options)`** — Same as above but for workflow steps. Automatically advances the workflow state machine after execution.

- **`HttpTransportManager`** — Implements `TransportManager`. Uses Redis for cache operations, HTTP POST (`fetch()`) for message dispatch. Supports optional HMAC signing via `secret`.

- **`WorkflowAdvancer`** — Handles workflow state machine advancement in serverless mode, including `onSuccess`/`onFailure` branching, workflow completion, and chained workflows.

**Example usage:**

```typescript
// lib/synkro.ts
import { createSynkroServerless } from "@synkro/next";

export const synkro = createSynkroServerless({
  connectionUrl: process.env.REDIS_URL!,
  baseUrl: process.env.NEXT_PUBLIC_APP_URL!,
  secret: process.env.SYNKRO_SECRET,
  workflows: [{ name: "OrderProcessing", steps: [...] }],
  handlerRoutes: [
    { eventType: "UserSignedUp", url: "/api/events/user-signed-up" },
    { eventType: "workflow:OrderProcessing:ValidateOrder", url: "/api/workflows/order/validate" },
  ],
});

// app/api/events/user-signed-up/route.ts
import { createEventHandler } from "@synkro/next";
import { synkro } from "@/lib/synkro";

export const POST = createEventHandler(synkro.client, {
  eventType: "UserSignedUp",
  transport: synkro.transport,
  secret: process.env.SYNKRO_SECRET,
  handler: async (ctx) => {
    await sendWelcomeEmail(ctx.payload.email);
  },
});
```

### FT-09: `publishAfterResponse()` helper

Wraps Next.js 15+ `after()` API for fire-and-forget event publishing that executes after the response is sent to the client.

```typescript
import { publishAfterResponse } from "@synkro/next";

export async function POST(request: Request) {
  publishAfterResponse(synkro, "AuditLog", { action: "created" });
  return Response.json({ ok: true }); // responds immediately
}
```

### IMP-05: Align SynkroClient with @synkro/core v0.15.0

Added missing methods to `SynkroClient`:

- `off(eventType, handler?)` — Unsubscribe event handlers
- `getWorkflowState(requestId, workflowName)` — Query workflow instance state
- `cancelWorkflow(requestId, workflowName)` — Cancel a running workflow

### IMP-06: Bump peer dependency

`@synkro/core` peer dependency updated from `^0.9.0` to `^0.15.0`. Added `ioredis: ^5.0.0` as optional peer dependency (required for serverless mode).

### Requires `@synkro/core` v0.15.0+

Core v0.15.0 added:
- `TransportManager` as a valid `transport` option (enables custom transports like `HttpTransportManager`)
- `executeHandler()` — extracted standalone handler execution with dedup/lock/retry/metrics (used by `createEventHandler` and `createWorkflowStepHandler`)
