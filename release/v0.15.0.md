# @synkro/core v0.15.0

**Release date:** 2026-03-09

## Changes

### TD-10: Unsubscribe / `off()` capability

Added `synkro.off(eventType, handler?)` to unregister event handlers at runtime. When a specific handler reference is passed, only that handler is removed. When called without a handler, all handlers for the event type are removed. When no handlers remain for an event type, the underlying transport channel subscription is cleaned up automatically.

### FT-04: Workflow state query API

Exposed `synkro.getWorkflowState(requestId, workflowName)` to query the current state of a workflow instance. Returns a `WorkflowState` object with `workflowName`, `currentStep`, and `status` fields, or `null` if no state exists. The `WorkflowState` type is now exported from `@synkro/core`.

### FT-07: Workflow cancellation

Added `synkro.cancelWorkflow(requestId, workflowName)` to cancel a running workflow. Sets the workflow status to `"cancelled"` and clears any active step timers. Returns `true` if the workflow was running and is now cancelled, `false` otherwise. Cancelled workflows will not advance to subsequent steps even if a step is mid-execution.

### IMP-01: Typed payload generics

`HandlerCtx`, `HandlerFunction`, and `SynkroEvent` are now generic types with a default of `unknown`. Handlers can opt into typed payloads via `HandlerCtx<{ name: string }>` for compile-time safety. Fully backward-compatible — existing code using unparameterized types continues to work unchanged.

### TD-09: Fix eslint-disable for decorator types

Replaced `Function` type constraint in `@OnEvent` and `@OnWorkflowStep` decorators with `(...args: any[]) => any`, eliminating the `@typescript-eslint/no-unsafe-function-type` eslint-disable comments.
