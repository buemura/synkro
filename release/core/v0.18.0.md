# @synkro/core v0.18.0

**Release date:** 2026-03-10

## Changes

### FT-02: Scheduled and delayed event publishing

Support for delayed and recurring event publishing:

- `synkro.publishDelayed(event, payload, delayMs)` — publishes an event after a one-shot delay, returns the requestId immediately
- `synkro.schedule(eventType, intervalMs, payload?)` — creates a recurring event publish on a fixed interval, returns a scheduleId
- `synkro.unschedule(scheduleId)` — cancels a scheduled recurring publish
- All active timers are automatically cleaned up on `synkro.stop()`
- Active schedules are included in `introspect().schedules`

### IMP-03: Middleware / interceptor pipeline

Koa-style middleware chain for cross-cutting concerns (logging, tracing, auth, validation):

- `synkro.use(middleware)` — registers a middleware function at runtime
- `SynkroOptions.middlewares` — register middlewares at startup
- Middleware signature: `(ctx: MiddlewareCtx, next: () => Promise<void>) => Promise<void>`
- `MiddlewareCtx` extends `HandlerCtx` with `eventType` so middleware knows which event it is processing
- Middlewares execute in registration order (onion model), wrapping each handler independently
- Zero overhead when no middleware is registered
- Works on both regular events and workflow steps
- Also supported in the standalone `executeHandler` for serverless environments
- New exports: `composeMiddleware`, `MiddlewareCtx`, `MiddlewareFunction`

### FT-10: Workflow visualization / DAG export

Expose workflow definitions as a graph structure for visualization:

- `synkro.getWorkflowGraph(workflowName)` — returns a `WorkflowGraph` or `null`
- `WorkflowGraph` contains `nodes` (one per step with optional retry/timeout metadata) and `edges` (sequential "next", "onSuccess", "onFailure")
- `introspect().graphs` returns all workflow DAGs
- New exported types: `WorkflowGraph`, `WorkflowGraphNode`, `WorkflowGraphEdge`
