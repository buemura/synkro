# @synkro/core v0.11.0

**Release date:** 2026-03-07

## Changes

### TD-08: Validate transport field [SEC]

`Synkro.start()` now explicitly validates the `transport` option. Only `"redis"`, `"in-memory"`, and `undefined` (defaults to Redis) are accepted. Previously, any invalid value silently fell back to Redis, potentially exposing events to a shared bus. Invalid values now throw a descriptive error.

### TD-04: Async publish with error propagation

**Breaking:** `TransportManager.publishMessage` signature changed from `void` to `Promise<void>`. Custom transport implementations must update their return type. `RedisManager` now awaits the underlying Redis `publish()` call, and all call sites (`Synkro.publish`, `HandlerRegistry`, `WorkflowRegistry`) properly await and propagate publish errors instead of silently swallowing them.

### TD-05: processingLocks observability

Added warning logs when `processingLocks` exceeds 1000 entries in both `HandlerRegistry` and `WorkflowRegistry`, enabling production monitoring of potential lock accumulation under high concurrency.

### IMP-02: Error details in failure events

Failure events now include an `errors` array with serialized error details:

```json
{
  "requestId": "...",
  "payload": { ... },
  "errors": [{ "message": "handler failed", "name": "Error" }]
}
```

Both `Error` instances (serialized with `message` and `name`) and non-Error thrown values (serialized with `message` only) are supported. Success/completion events remain unchanged.

### TD-12: Remove dead code

Removed the unused `eventToWorkflows` map from `WorkflowRegistry` that was populated during registration but never read.
