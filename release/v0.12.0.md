# @synkro/core v0.12.0

**Release date:** 2026-03-07

## Changes

### FT-14: Implicit step registration for onSuccess/onFailure targets

Workflow steps referenced by `onSuccess` or `onFailure` no longer need to be explicitly declared in the `steps` array. The workflow engine now automatically appends missing branch targets as implicit steps during registration.

**Before:**
```typescript
{
  name: "OrderProcessing",
  steps: [
    { type: "ValidateOrder" },
    { type: "ProcessPayment", onFailure: "HandlePaymentFailure" },
    { type: "FulfillOrder" },
    { type: "HandlePaymentFailure" }, // required but redundant
  ],
}
```

**After:**
```typescript
{
  name: "OrderProcessing",
  steps: [
    { type: "ValidateOrder" },
    { type: "ProcessPayment", onFailure: "HandlePaymentFailure" },
    { type: "FulfillOrder" },
    // HandlePaymentFailure is auto-registered
  ],
}
```

This change is fully backward compatible — explicitly declaring branch target steps still works as before.
